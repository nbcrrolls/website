LigandSimulation Workflow

Required software:

  • Autodock vina 1.1.2 and above
  • VMD 1.9.1 and above
  • AmberTools 12 and above
  • Linux platform

Notes

Caution: This workflow is designed for the users who have the background information on AmberTools, especially tleap, VMD and Autodock vina. Because many steps are automated, there is a specific naming scheme for the input files. Required input files are:

  • ligand-new.pdb
  • receptor.pdbqt
  • receptor-noh.pdb
  • ligand.prepc
  • ligand.frcmod
ligand and receptor here are variables that must match the ligand and receptor parameters. All these input files are also the output files of the Ligand Parameterization workflow.

This workflow was originally created for a project that involved a zinc ion. As a result, when running this workflow, it calls for a zinc parameter file (frcmod.zinc) and three zinc libraries (znb.lib, hin.lib and hydroxide.lib) that could be downloaded from this site. These files are not necessary if user's project does not involve zinc ion. Users will need to delete the following lines, which can be found in PDB Modification actor in String Constant 4:

  • loadamberparams /scratch/pieong/zinc_reduced_set/frcmod.zinc
  • loadoff /scratch/pieong/zinc_reduced_set/znb.lib
  • loadoff /scratch/pieong/zinc_reduced_set/hin.lib
  • loadoff /scratch/pieong/zinc_reduced_set/hydroxide.lib
User can open PDB modification by right-clicking on PDB Modification actor and selecting Open actor from a menu.
Once inside the actor, double click on String Constant 4 and make edits in a pop-up window.

The output files from simulations are not automatically downloaded from a remote host to a local machine.
